# MVP Scope

Status: Canonical
Scope: First implementation boundary
Authority: High
Last reviewed: 2026-05-26

## Goal

最初の目標は、日常的にMarkdownファイルを開き、編集し、保存できる最小エディタを作ることです。

この段階では、Markdown執筆の基本体験と安全なファイルI/Oの境界を固めます。

## Prototype State

2026-05-26時点で、Tauri + React + CodeMirror 6による触れるプロトタイプは存在する。

現在できること:

- ファイルピッカーからテキスト/Markdownファイルを選ぶ
- UTF-8テキストとして開く
- CodeMirrorで編集する
- Rust command経由で保存する
- Markdownプレビューを表示する
- バイナリらしいファイルを拒否する
- 5MB以上を警告扱いにする
- 10MB超をプロトタイプ編集対象から外す

この段階の次の課題は、機能拡張ではなく保存と外部変更検知の堅牢化。

タブとファイルツリーは重要だが、単一ファイル保存の安全性を固めてから入れる。

## In Scope

- 単一ファイルを開く
- UTF-8テキストを表示する
- Markdownシンタックスハイライト
- プレーンテキスト編集
- 保存
- Cmd+S保存
- 未保存状態の表示
- 外部変更の簡易検知
- 大容量ファイルの警告
- バイナリファイルの読み込み防止

## Out of Scope

この一覧は初期MVPの境界を示す。プロトタイプではMarkdownプレビューまで実装済み。

- フォルダツリー
- 複数タブ
- Markdown lint
- Diff表示
- Git連携
- AI支援
- 設定画面
- プラグイン機構

## Acceptance Criteria

- `.md` ファイルをファイルピッカーから開ける
- 開いた本文を編集できる
- Cmd+Sで同じファイルへ保存できる
- 未保存変更が画面上で分かる
- バイナリらしいファイルは編集画面で開かない
- 5MB以上のファイルは開く前に警告する
- アプリから外部コマンドを実行する経路がない

## First Slice

最初の実装単位は以下に絞る。

1. Tauriアプリの最小起動
2. CodeMirror 6のエディタ表示
3. ファイルを1つ開く
4. UTF-8として読み込む
5. 保存する
6. 保存前後の状態を表示する

ここまでできてから、保存安全性、フォルダツリー、タブへ進む。
