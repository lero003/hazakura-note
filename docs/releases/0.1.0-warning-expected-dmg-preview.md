## Proposed Release Note Addendum

This is a warning-expected DMG preview. It is not the source-only developer preview release.

The preview is for users who want to inspect the locally built app without running the build steps themselves.

The DMG is a developer-preview convenience artifact. It is ad-hoc signed only. It is not Developer ID signed, not notarized, and macOS may show security warnings or reject normal Gatekeeper assessment.

Only use this preview if you are comfortable evaluating unsigned and not-notarized macOS preview builds.

Artifact:

```txt
hazakura-note_0.1.0_aarch64-warning-expected.dmg
```

Platform:

```txt
macOS Apple Silicon / aarch64
```

SHA-256:

```txt
68d45b3699e4f3f3c529448bdbdbe6e24bb541586be4e3cbbf2a74d4104a716a
```

Local verification completed before publication approval:

```bash
npm ci
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build:dmg-preview
npm audit
cargo audit
hdiutil verify src-tauri/target/release/bundle/dmg/hazakura-note_0.1.0_aarch64-warning-expected.dmg
shasum -c src-tauri/target/release/bundle/dmg/hazakura-note_0.1.0_aarch64-warning-expected.dmg.sha256
```

The DMG was mounted locally. The contained `hazakura-note.app` launched from the mounted volume, showed a window, exposed native File and View menus, and kept File > Save disabled in the no-file state.

`cargo audit` completed with warnings from transitive Tauri/wry dependencies, mostly GTK3/gtk-rs Linux stack unmaintained advisories plus `unic-*` and `glib` warnings. No pasted high or critical vulnerability finding is treated as a blocker for this macOS warning-expected DMG preview, but the warnings should stay visible for dependency-update review.

## Required Publication Caveat

Do not describe this artifact as signed for public distribution, notarized, trusted, or installer-grade.

Do not attach this DMG to a release until the user explicitly approves binary asset publication.

## Known Packaging Note

The warning-expected preview lane uses `npm run build:dmg-preview`, which creates a plain DMG directly with `hdiutil create` from the freshly built app bundle. This avoids the Finder/`osascript` layout step used by the Tauri create-dmg path, which stalled in the local automation environment.

Treat this as acceptable only for the warning-expected preview lane. A Developer ID / notarized DMG lane needs separate signing, hardened runtime, notarization, stapling, and Gatekeeper verification work.
